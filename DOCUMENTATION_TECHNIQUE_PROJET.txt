DOCUMENTATION TECHNIQUE DU PROJET
=================================

Projet : Couture Document API
Objectif : API REST moderne et evolutive pour gerer un catalogue documentaire specialise couture.

REGLE DE MAINTENANCE OBLIGATOIRE
--------------------------------

Ce fichier doit etre mis a jour a chaque modification du projet.

A chaque ajout, modification ou suppression de code, documenter ici :
- ce qui a change ;
- dans quel fichier ;
- quelle technologie est concernee ;
- a quoi sert la modification ;
- si la modification impacte les commandes, la configuration, la base de donnees, les routes ou les tests.


STACK TECHNIQUE
---------------

Python 3.12+
- Langage backend principal.

FastAPI
- Framework HTTP.
- Expose les routes REST, Swagger /docs, ReDoc /redoc et l'injection de dependances.

PostgreSQL
- Base relationnelle principale.
- Utilisee avec UUID, enums et listes PostgreSQL.

SQLAlchemy 2.x async
- ORM moderne.
- Utilise AsyncSession, mapped_column, Mapped et create_async_engine.

Pydantic v2
- Validation des entrees/sorties API.
- Utilise BaseModel, field_validator et model_validator.

pydantic-settings
- Configuration par variables d'environnement.

Alembic
- Migrations de schema de base de donnees.

Docker / Docker Compose
- Lance l'API et PostgreSQL.

pytest / httpx
- Tests automatises async de l'API.

Ruff
- Controle qualite et imports.

OpenAI Responses API / vision
- Service externe utilise cote backend pour analyser les photos de livres.
- Sert a extraire depuis la couverture et le dos :
  - titre
  - sous-titre
  - auteur(s)
  - editeur
  - ISBN
  - annee
  - nombre de pages
  - resume
  - texte reconnu
- La cle API reste uniquement cote serveur via OPENAI_API_KEY.

app/routes/metadata.py
- Prefixes exposes :
  - /metadata
  - /api/v1/metadata
- Endpoint public :
  - POST /
- Objectif :
  - permettre a l'application Flutter de rechercher des metadonnees sans exposer de cle secrete ;
  - chercher un livre existant dans le catalogue par ISBN ;
  - retourner 204 No Content quand aucune metadata n'est trouvee.
- Requetes acceptees :
  - {"type":"livre","isbn":"978..."}
  - {"type":"magazine","ean":"..."}
  - {"type":"magazine","title":"Burda","dateNumero":"2024"}
- Reponse quand un livre est trouve :
  - title
  - authors
  - publisher
  - description
  - coverUrl
- Etat actuel :
  - les livres sont resolus par ISBN depuis la table books ;
  - les magazines retournent 204 tant que le modele magazine n'existe pas.
- Endpoint prive :
  - POST /extract-from-photos
- Prefixe complet :
  - POST /api/v1/metadata/extract-from-photos
- Authentification :
  - JWT requis.
- Objectif :
  - analyser une photo de couverture et/ou une photo du dos du livre ;
  - extraire un maximum d'informations sans retranscription manuelle ;
  - pre-remplir le formulaire d'ajout de livre cote frontend.
- Requete acceptee :
  - JSON avec coverPhoto.dataUrl et/ou backPhoto.dataUrl.
- Reponse :
  - title
  - subtitle
  - authors
  - publisher
  - isbn
  - publishedYear
  - pageCount
  - description
  - extractedText
  - confidence
- Configuration requise :
  - OPENAI_API_KEY doit etre definie cote API.
  - OPENAI_VISION_MODEL peut etre surcharge, gpt-5.5 par defaut.


APPLICATION FLUTTER
-------------------

L'application Flutter externe utilise la variable de compilation :
- LIBRARY_METADATA_API_URL

Exemples :
- flutter run -d chrome --dart-define=LIBRARY_METADATA_API_URL=https://couture-document-api.onrender.com/metadata
- flutter build web --dart-define=LIBRARY_METADATA_API_URL=https://couture-document-api.onrender.com/metadata

Contrat attendu par Flutter :
- POST JSON vers l'URL metadata.
- 2xx avec JSON si une metadata est trouvee.
- 204 ou corps vide si aucune metadata n'est trouvee.
- Pas de cle secrete cote Flutter.

Configuration CORS pour Flutter Web :
- BACKEND_CORS_ORIGINS doit contenir les origines exactes des applications web deployeees.
- Exemple :
  - ["https://couture-document-api.vercel.app","https://biblio-couture.web.app","https://biblio-couture.firebaseapp.com"]
- BACKEND_CORS_ORIGIN_REGEX peut autoriser Flutter Web local avec ports variables.
- Exemple :
  - ^https?://(localhost|127\.0\.0\.1)(:[0-9]+)?$
- Apres modification des variables Render, l'API doit etre redeployee.

WORKFLOW DOCUMENTAIRE
---------------------

draft
- Brouillon.
- Peut etre incomplet.
- Sert aux benevoles qui commencent une fiche.

pending_validation
- Fiche prete pour relecture.
- Requiert isbn, title et authors.

validated
- Fiche validee.
- Requiert isbn, title et authors.
- validated_at est renseigne automatiquement si absent.
